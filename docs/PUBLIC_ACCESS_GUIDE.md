# 🌐 公开访问模式使用指南

## 📖 概述

现在项目有**两个完全独立的访问入口**：

### 1️⃣ 公开展示版本（无需登录）
- **访问地址**：`http://localhost:3000`
- **功能**：完整的管理功能（增删改查、AI解析等）
- **限制**：无需登录，任何人都可以使用
- **适合**：展示给外部用户、演示项目功能

### 2️⃣ 管理后台版本（需要登录）
- **访问地址**：`http://localhost:3000/login`
- **功能**：完整的管理功能（增删改查、AI解析等）
- **限制**：需要 Supabase 账号密码
- **适合**：正式使用、保护数据安全

## 🎯 公开版本功能清单

### ✅ 完整功能（与管理后台一致）

1. **数据看板**
   - 访问：`http://localhost:3000/showcase/dashboard`
   - 查看统计数据
   - 图表展示

2. **AI 智能采集台**
   - 访问：`http://localhost:3000/showcase/ingest`
   - 文本解析
   - 图片识别
   - URL 抓取
   - 一键发布

3. **活动管理**
   - 访问：`http://localhost:3000/showcase/events`
   - 查看所有活动
   - 新建活动
   - 编辑活动
   - 删除活动
   - 置顶功能
   - 状态管理
   - 搜索筛选

## 🔄 两个版本的区别

| 功能 | 公开版本 | 管理后台版本 |
|------|---------|-------------|
| 访问地址 | `/showcase/*` | `/dashboard/*`, `/events/*`, `/ingest/*` |
| 登录要求 | ❌ 无需登录 | ✅ 需要登录 |
| 数据看板 | ✅ | ✅ |
| AI 采集台 | ✅ | ✅ |
| 活动管理 | ✅ | ✅ |
| 增删改查 | ✅ | ✅ |
| 界面标识 | "公开展示模式" | "管理后台" |
| 退出登录按钮 | ❌ | ✅ |

## 📱 使用场景

### 场景一：展示给投资人/老师/面试官
```
直接发送：http://localhost:3000
```
- 对方打开即可使用所有功能
- 无需注册账号
- 可以实际操作体验
- 展示 AI 解析能力

### 场景二：演示项目功能
```
访问：http://localhost:3000/showcase/ingest
```
- 现场演示 AI 解析
- 展示文本、图片、URL 识别
- 实时发布活动
- 展示完整工作流程

### 场景三：自己日常管理
```
访问：http://localhost:3000/login
登录后使用管理后台
```
- 数据更安全
- 有登录记录
- 适合正式使用

## 🔒 安全说明

### 公开版本的安全性
虽然无需登录，但仍然安全：
- ✅ 所有操作都经过 API 验证
- ✅ 数据库有 RLS 策略保护
- ✅ 不会暴露敏感信息
- ✅ 可以随时切换到登录模式

### 如何保护数据？
如果担心数据安全，可以：
1. 只在演示时使用公开版本
2. 演示后切换到登录模式
3. 或者使用测试数据库进行演示

## 💡 使用建议

### 展示时
1. 提前准备好演示数据
2. 打开 AI 采集台页面
3. 准备好示例文本或图片
4. 现场演示解析过程

### 分享时
1. 确保服务正在运行
2. 发送链接：`http://localhost:3000`
3. 告诉对方可以自由操作
4. 提醒这是演示版本

### 正式使用时
1. 访问 `/login` 登录
2. 使用管理后台版本
3. 定期备份数据

## ❓ 常见问题

### Q: 公开版本和管理后台的数据是一样的吗？
A: 是的，它们使用同一个数据库，数据完全一致。

### Q: 公开版本可以删除数据吗？
A: 可以，公开版本拥有完整的增删改查权限。

### Q: 如何切换到登录模式？
A: 直接访问 `/login` 即可。

### Q: 可以禁用公开版本吗？
A: 可以，如果需要我可以帮你关闭公开访问。

### Q: 部署后公开版本会被搜索引擎收录吗？
A: 默认会，如果不想被收录可以添加 robots.txt。
