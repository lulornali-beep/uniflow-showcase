# 只读展示模式使用说明

## 📖 概述

项目现在有两个访问模式：

### 1. 公开展示模式（只读）
- **访问地址**：`http://localhost:3000` 或 `http://localhost:3000/showcase`
- **无需登录**：任何人都可以直接访问
- **只能查看**：只能浏览活动信息，不能修改
- **适合场景**：展示给外部用户、分享成果

### 2. 管理后台模式（可编辑）
- **访问地址**：`http://localhost:3000/login`
- **需要登录**：需要 Supabase 账号密码
- **完整权限**：可以增删改查所有数据
- **适合场景**：管理员日常管理

## 🎯 使用场景

### 场景一：展示给朋友/老师/投资人
1. 直接发送链接：`http://localhost:3000`
2. 对方打开即可浏览所有已发布的活动
3. 对方无法修改任何数据

### 场景二：自己管理内容
1. 访问：`http://localhost:3000/login`
2. 输入 Supabase 账号密码登录
3. 进入管理后台进行编辑

## 🔒 安全说明

### 展示模式的安全性
- ✅ 只能查看已发布的活动（status = 'published'）
- ✅ 无法看到草稿和已下架的内容
- ✅ 无法修改、删除任何数据
- ✅ 无法访问管理后台功能
- ✅ 数据库层面的保护（RLS 策略）

### 管理模式的安全性
- 🔐 需要 Supabase 账号密码
- 🔐 登录后才能访问管理功能
- 🔐 可以管理所有数据

## 🌐 部署到线上

如果你想把项目部署到互联网上，让任何人都能访问：

2. 连接 GitHub 仓库
3. 一键部署

### 访问方式
- 公开展示：`https://your-project.vercel.app`
- 管理后台：`https://your-project.vercel.app/login`
